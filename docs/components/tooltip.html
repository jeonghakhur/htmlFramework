Usage

툴팁 플러그인은 요청 시 콘텐츠 및 마크업을 생성하며 기본적으로 툴팁을 트리거 요소 뒤에 배치합니다.

JavaScript를 통해 툴팁 트리거:

Overflow auto and scroll

툴팁위치는 상위 컨테이너가 overflow: auto 이거나 overflow: scroll 일 경우 원래 지정한 위치를 무시하고 자동으로 위치를 배치합니다. 기본값을 유지하려면 boundary 옵션값을 'window'로 변경하세요

Markup

툴팁에 필요한 마크업은 툴팁을 추가 정보를 가질 HTML 요소에 대한 데이터 속성 및 제목입니다. 생성된 툴팁의 마크업은 위치는 body 요소의 바로 자식요소로 생성됩니다.

키보드 및 보조 기술 사용자에게 툴팁 제공

전통적으로 키보드 포커스가 가능하고 대화형(링크 또는 양식 컨트롤 등)인 HTML 요소에만 툴팁을 추가해야 합니다. span과 같은 임의 HTML 요소는 tabindex="0" 특성을 추가하여 포커스가 가능하도록 만들 수 있지만, 이것은 키보드 사용자를 위한 비인터랙티브 요소에 잠재적으로 성가시고 혼란스러운 탭을 추가할 수 있습니다.

또한 툴팁을 트리거로 hover를 사용하지 마십시오. 키보드 사용자에게는 툴팁을 트리거할 수 없습니다.

Disabled elements

disabled 속성이있는 요소는 대화 형이 아니므로 사용자가 초점을 맞추거나 마우스를 가져 가거나 클릭하여 툴팁 (또는 팝 오버)을 트리거 할 수 없습니다. 해결 방법으로, tabindex = "0"을 사용하여 키보드 초점을 맞출 수있는 래퍼 <div> 또는 <span>에서 툴팁을 트리거하고 비활성화 된 요소의 포인터 이벤트를 재정의 할 수 있습니다.
