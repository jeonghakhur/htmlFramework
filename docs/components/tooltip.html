Usage

툴팁 플러그인은 요청 시 콘텐츠 및 마크업을 생성하며 기본적으로 툴팁을 트리거 요소 뒤에 배치합니다.

JavaScript를 통해 툴팁 트리거:

Overflow auto and scroll

툴팁위치는 상위 컨테이너가 overflow: auto 이거나 overflow: scroll 일 경우 원래 지정한 위치를 무시하고 자동으로 위치를 배치합니다. 기본값을 유지하려면 boundary 옵션값을 'window'로 변경하세요

Markup

툴팁에 필요한 마크업은 툴팁을 추가 정보를 가질 HTML 요소에 대한 데이터 속성 및 제목입니다. 생성된 툴팁의 마크업은 위치는 body 요소의 바로 자식요소로 생성됩니다.

키보드 및 보조 기술 사용자에게 툴팁 제공

전통적으로 키보드 포커스가 가능하고 대화형(링크 또는 양식 컨트롤 등)인 HTML 요소에만 툴팁을 추가해야 합니다. span과 같은 임의 HTML 요소는 tabindex="0" 특성을 추가하여 포커스가 가능하도록 만들 수 있지만, 이것은 키보드 사용자를 위한 비인터랙티브 요소에 잠재적으로 성가시고 혼란스러운 탭을 추가할 수 있습니다.

또한 툴팁을 트리거로 hover를 사용하지 마십시오. 키보드 사용자에게는 툴팁을 트리거할 수 없습니다.

Disabled elements

disabled 속성이있는 요소는 대화 형이 아니므로 사용자가 초점을 맞추거나 마우스를 가져 가거나 클릭하여 툴팁 (또는 팝 오버)을 트리거 할 수 없습니다. 해결 방법으로, tabindex = "0"을 사용하여 키보드 초점을 맞출 수있는 래퍼 <div> 또는 <span>에서 툴팁을 트리거하고 비활성화 된 요소의 포인터 이벤트를 재정의 할 수 있습니다.

Options

옵션은 데이터 속성 또는 JavaScript를 통해 전달할 수 있습니다. 데이터 속성의 경우 data-animation = ""에서와 같이 data-에 옵션 이름을 추가합니다.

보안상의 이유로 sanitize, sanitizeFn 및 whiteList 옵션은 데이터 속성을 사용하여 제공 할 수 없습니다.

Name Type Default Description
animation boolean true 툴팁에 CSS fade transition 적용
container string | element | false false

특정 요소에 툴팁을 추가합니다. 예: container: 'body'. 이 옵션은 트리거 요소 근처의 문서 흐름에서 도구 설명을 배치 할 수 있다는 점에서 특히 유용합니다.
delay number | object 0

툴팁 표시 및 숨기기 지연 (ms)-수동 트리거 유형에는 적용되지 않습니다.

숫자를 입력하면 숨기기 / 보이기 모두에 지연이 적용됩니다.

Object structure is: delay: { "show": 500, "hide": 100 }
html boolean false

툴팁에서 HTML을 허용합니다.

true 인 경우 툴팁의 title 에있는 HTML 태그가 툴팁에 렌더링됩니다.
placement string | function 'top'

툴팁을 배치하는 방법 - auto | top | bottom | left | right.
auto 가 지정되면 툴팁의 방향이 동적으로 변경됩니다.
selector string | false false 선택기가 제공되면 툴팁 개체가 지정된 대상에 위임됩니다. 실제로 이것은 동적으로 추가 된 DOM 요소에 툴팁을 적용하는데도 사용됩니다.
template string '<div class="tooltip" role="tooltip"><div class="arrow"></div><div class="tooltip-inner"></div></div>'

툴팁을 만들 때 사용할 기본 HTML입니다.

툴팁의 title 이 .tooltip-inner 에 삽입됩니다..

.arrow가 툴팁의 화살표가됩니다.

가장 바깥 쪽 래퍼 요소에는 .tooltip 클래스와 role = "tooltip"이 있어야합니다.
title string | element | function ''

title 속성이없는 경우 기본 제목 값입니다.

함수가 제공되면 툴팁이 첨부 된 요소로 설정된 this 참조와 함께 호출됩니다.
trigger string 'hover focus'

툴팁이 트리거되는 방법 - click | hover | focus | manual. 여러 트리거를 전달할 수 있습니다. 공백으로 구분하십시오.

'manual' 툴팁이 다음을 통해 프로그래밍 방식으로 트리거됨을 나타냅니다..tooltip('show'), .tooltip ( 'hide') 및 .tooltip ( 'toggle') 이 값은 다른 트리거와 결합 할 수 없습니다.
offset number | string | function 0

대상을 기준으로 한 툴팁의 오프셋입니다.

오프셋을 결정하기 위해 함수를 사용하면 오프셋 데이터를 첫 번째 인수로 포함하는 객체를 사용하여 호출됩니다. 함수는 동일한 구조를 가진 객체를 반환해야합니다. 트리거 요소 DOM 노드는 두 번째 인수로 전달됩니다.
boundary string | element 'scrollParent' 'viewport', 'window', 'scrollParent'
개별 툴팁에 대한 데이터 속성

위에서 설명한대로 데이터 속성을 사용하여 개별 툴팁에 대한 옵션을 지정할 수도 있습니다.

Methods

비동기 메서드 및 전환

모든 API 메서드는 비동기식이며 전환을 시작합니다. 전환이 시작 되 자마자 종료되기 전에 발신자에게 돌아갑니다. 또한 전환 구성 요소에 대한 메서드 호출이 무시됩니다.

$().tooltip(options)

요소 컬렉션에 도구 설명 처리기를 연결합니다.

.tooltip('show')

요소의 툴팁을 표시합니다.

.tooltip('hide')

요소의 툴팁을 숨 깁니다.

.tooltip('toggle')

요소의 툴팁을 전환합니다.

.tooltip('dispose')

요소의 툴팁기능을 제거합니다.

.tooltip('enable')

요소의 툴팁이 표시 할 수있는 기능을 제공합니다. 툴팁은 기본적으로 활성화되어 있습니다.

.tooltip('disable')

요소의 툴팁이 표시되는 기능을 제거합니다. 툴팁은 다시 활성화 된 경우에만 표시 될 수 있습니다.

.tooltip('toggleEnabled')

요소의 툴팁을 표시하거나 숨기는 기능을 토글합니다.

.tooltip('update')

요소의 툴팁 위치를 업데이트합니다.

Event

Event type Description
show.wbm.tooltip 이 이벤트는 show instance 메서드가 호출되면 즉시 발생합니다.
shown.wbm.tooltip 이 이벤트는 사용자가 툴팁을 볼 수 있을 때 발생합니다(CSS 전환이 완료될 때까지 기다립니다).
hide.wbm.tooltip 이 이벤트는 인스턴스 숨기기 메서드가 호출되면 즉시 실행됩니다.
hidden.wbm.tooltip 이 이벤트는 툴팁이 사용자로부터 숨겨질 때 발생합니다(CSS 전환이 완료될 때까지 대기).
inserted.wbm.tooltip 이 이벤트는 툴팁 템플릿이 DOM에 추가된 경우 show.wbm.tooltip 이벤트 후에 발생합니다.
